# 端口规划

本文只使用示例中继服务器地址 `192.0.2.44` 和公开示例端口。真实中继服务器地址、端口和主机映射由部署者自行配置。

## 当前架构

```text
管理端 / OpenSSH                 私有 中继服务器 / Linux                 Windows
执行 curl 或 SSH  ───────────→  Chisel Server                 Agent 127.0.0.1:18888
                                 反向端口 19101  ←──────────  Chisel client
```

## 示例端口

| 端口 | 用途 | 位置 |
|------|------|------|
| 28271 | Chisel Server | 中继服务器 |
| 18090 | 可选的部署文件服务 | 中继服务器，建议仅内网或临时启用 |
| 19101、19102…… | Windows Agent 反向端口 | 中继服务器，优先只监听 localhost |
| 18888 | Windows Agent | Windows `127.0.0.1` |

## 示例主机映射

以下主机名和端口均为虚构示例，不对应任何实际部署：

```text
win-lab-a → Windows 127.0.0.1:18888 → 中继服务器 127.0.0.1:19101
win-lab-b → Windows 127.0.0.1:18888 → 中继服务器 127.0.0.1:19102
win-test-c → Windows 127.0.0.1:18888 → 中继服务器 127.0.0.1:19103
```

## 调用规则

优先 SSH 登录 中继服务器 后访问本机回环地址：

```bash
ssh <user>@<vps-host>
curl -sS http://127.0.0.1:19101/health
```

正式任务使用 `/stdin-run` 并携带 Agent Token。不要把真实 Token 写入文档。
