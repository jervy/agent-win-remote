# 安全策略

## 使用范围

Agent Win Remote 只能用于你拥有或明确获授权的 Windows 设备。它具备远程 PowerShell 执行能力，错误配置可能导致严重后果。

## 不要公开的内容

请不要在 Issue、Pull Request、日志或公开仓库中提交：

- 中继服务器地址、SSH 私钥或密码
- Chisel 用户名和密码
- Agent Token
- 真实 Windows 主机名、局域网 IP 和端口映射
- 含个人数据的执行结果

## 报告安全问题

不要在公开 Issue 中粘贴可用凭据、完整攻击步骤或真实设备信息。

如果仓库已启用 GitHub Security Advisories，请优先通过私下 Advisory 报告；否则请联系仓库维护者，说明：

- 受影响的文件和版本
- 可复现条件
- 影响范围
- 建议修复方式

提交报告前请删除所有真实 Token、密码、IP 和个人数据。

## 当前安全边界

- Windows Agent 默认只监听 `127.0.0.1`
- 远程管理接口要求 Bearer Token
- 请求体、脚本执行时间和输出大小有限制
- 不安装服务，不写注册表自启动，不做持久化
- 不提供交互式终端、PTY、WebSocket shell 或专用提权接口
- Chisel 认证、SSH 认证和 中继服务器防火墙需要由部署者正确配置

## 凭据泄露处理

如果 Token 或密码曾经提交到 Git：

1. 立即撤销并重新生成凭据
2. 修改 中继服务器防火墙和 SSH 密钥
3. 清理 Git 历史中的敏感内容
4. 检查 Agent 日志和 中继服务器 登录日志
5. 不要只删除当前文件后继续使用旧凭据
